Reviewing

ReviewMe provides a simple way to allow users of your guild to review each other's work, transactions, or otherwise.

Portfolios

Alongside reviews, users can make basic portfolios that can include images and discord markdown formatting.

Customization

Furthermore, admins can customize the embed format, changing the title, body, 3 fields, and the footer's contents all using templates to stylize it more based on the guild.

Administration

Finally, admins can remove reviews, add cooldowns, manage which roles can be reviewed or can review at all, and change message styling.

Quick Setup

These speedy steps will help get you on your merry way!

Setup Roles

Roles provide a way to seperate reviewees (the provider) from reviewers (the client/customer). In some cases, you may want to only allow certain roles to be reviewed, or only allow certain roles to review at all.

To configure this, use /admin role reviewee to add users that can be reviewed, and use /admin role reviewer to change who can create reviews (or alternatively, just remove permissions to run the /review command as described later)

By default, no roles are required to review or be reviewed.

Setup Commands

Slash Commands are an amazing discord feature in such that: you can configure where you want commands to be ran. To change this, navigate to Server Settings > Integrations > ReviewMe to change where commands can be run, and by who (for role locking). This is for the case where you might have a #reviews channel or likewise.

Setup Embed

Embeds are the magic behind good looking experiences, and you can configure your own using /admin setting embed [section].

The embeds use a "templating" system, where-as you provide tokens such as {reviewee} and once it renders the embed, it will replace those tokens with actual data. Here's a list of all available tokens:
  • {reviewee} - The reviewee's nickname
  • {reviewer} - The reviewer's nickname
  • {@reviewee} - The reviewee's mention
  • {@reviewee} - The reviewer's mention
  • {rating} - The reviewer's rating as 5 star emojis and a number
  • {service} - The reviewer's product/service
  • {review} - The reviewer's review

Privacy Policy

ReviewMe and its creators, namely Command_Storm (also known as AfterAStorm), may collect data about guilds or their members only for the usage of the bot.

This policy can also be viewed through the /about slash command.

Information collected

ReviewMe and its creators collects the following information (formatted as "Definition" and "Purpose"):
  • User Id
    • A way to differenciate Discord users.
    • Used for: storing reviews, storing portfolios
  • Guild Id
    • A way to differenciate Discord guilds.
    • Used for: storing reviews, storing portfolios, storing settings
  • Role Id
    • A way to differenciate Discord roles.
    • Used for: storing settings
Reviews and other user-input text is not encrypted, but ideally you, a user, wouldn't include any personable or regrettable data within.

Data removal

When a guild is deleted, the bot is kicked, or the bot leaves on its own, (all of this assuming it's online) all data associated with the guild will be lost and forgotten (including reviews). For user data removal, contact the developer from the Discord support server located here.
